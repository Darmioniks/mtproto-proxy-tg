MTProto Proxy Web Manager
Веб-приложение на языке программирования Python для работы с публичными прокси-серверами Telegram (MTProto). Программа автоматически загружает актуальный список прокси из удаленного репозитория GitHub, проверяет их доступность с помощью быстрого асинхронного TCP-пинга и выводит результаты в виде компактной интерактивной таблицы.

Интерфейс разработан на фреймворке Streamlit, что позволяет развернуть полноценный сайт и использовать его с любых устройств, включая смартфоны и планшеты.

Структура проекта
app.py — Главный файл веб-приложения со всей логикой интерфейса и сети.

requirements.txt — Файл со списком внешних зависимостей для сборки проекта на хостинге.

Установка и запуск локально
Для запуска приложения на локальном компьютере выполните следующие шаги.

1. Установка зависимостей
Убедитесь, что у вас установлен Python версии 3.9 или выше. Установите необходимые библиотеки:

Bash
pip install streamlit requests
Или используйте файл зависимостей:


pip install -r requirements.txt

2. Запуск локального сервера
Запустите веб-приложение через встроенный движок Streamlit:


streamlit run app.py

После этого в браузере автоматически откроется вкладка с интерфейсом программы (по умолчанию адрес http://localhost:8501). Для доступа других устройств из вашей локальной Wi-Fi сети используйте адрес, указанный в строке Network URL в терминале.

Развертывание на бесплатном хостинге (Streamlit Cloud)
Сайт можно бесплатно опубликовать в сети Интернет, чтобы иметь к нему доступ из любого места без включенного компьютера.

Создайте публичный репозиторий на GitHub и загрузите туда файлы app.py и requirements.txt.

Авторизуйтесь на платформе Streamlit Community Cloud с помощью своего аккаунта GitHub.

Нажмите кнопку New app, выберите ваш репозиторий, ветку (main или master) и укажите путь к главному файлу (app.py).

Нажмите Deploy!. Через 1–2 минуты приложение будет доступно по постоянной публичной ссылке.

Функциональные особенности
Компактная табличная сетка: Данные отображаются в плотном структурированном виде, адаптированном под экраны мобильных телефонов. Каждая строка содержит порядковый номер, замер отклика и прямую ссылку.

Цветовая индикация: Активные прокси-серверы выделяются зеленым цветом с указанием точного пинга в миллисекундах.

Прямое подключение (Deep Linking): Ссылки автоматически преобразуются из веб-формата в системный протокол tg://. При клике на ссылку устройство пользователя мгновенно открывает установленное приложение Telegram с окном добавления прокси, минуя промежуточные страницы в браузере.

Случайная выборка: Программа перемешивает исходный список прокси перед проверкой, обеспечивая разнообразие доступных серверов при каждом обновлении.

Техническое описание работы сети
Проверка доступности серверов реализована через замер времени установления низкоуровневого TCP-соединения с IP-адресом и портом прокси-сервера. Таймаут ожидания ограничен 0.4 секундами для обеспечения высокой скорости работы веб-интерфейса.

Обратите внимание, что при запуске приложения на удаленном хостинге пинг измеряется между сервером хостинга и прокси-сервером, а не между конечным устройством пользователя и прокси. Данный метод служит первичным фильтром для отсеивания неактивных серверов. Финальная проверка криптографического протокола MTProto и секретного ключа (secret) происходит непосредственно внутри клиента Telegram при попытке подключения.
